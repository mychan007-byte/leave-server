# ระบบการลาราชการ — สถานีตำรวจภูธรดอนหัวฬ่อ (เซิร์ฟเวอร์กลาง)

ระบบยื่นใบลา/อนุมัติแบบหลายระดับ ใช้ร่วมกันทั้งสถานีผ่านเครือข่าย/อินเทอร์เน็ต
ข้อมูลรวมศูนย์ที่เซิร์ฟเวอร์เดียว (ไฟล์ `data/db.json`)

## คุณสมบัติ
- เข้าสู่ระบบด้วยการเลือกชื่อจากทำเนียบกำลังพล (ไม่ต้องใช้รหัส)
- ยื่นใบลา 3 ประเภท: ลาป่วย / ลากิจส่วนตัว / ลาพักผ่อน พร้อมนับวันและเช็คสิทธิคงเหลือ
- อนุมัติตามลำดับชั้น: หัวหน้างาน → รอง ผกก. → ผกก. (กำหนดสายอนุมัติตามสังกัดอัตโนมัติ)
- หน้าผู้ดูแลระบบ (ป้องกันด้วยรหัสผ่าน): จัดการรายชื่อ/สิทธิ์, ตั้งสายการอนุมัติ, นำเข้า Excel, รายงาน
- นำเข้ารายชื่อจากไฟล์ Excel (ทำเนียบกำลังพล) — กำหนดสิทธิ์/สายอนุมัติให้อัตโนมัติ
- พิมพ์ใบลาราชการ และรายงานสรุป เป็น PDF (ผ่านหน้าพิมพ์ของเบราว์เซอร์ รองรับภาษาไทยเต็มรูปแบบ)
- แจ้งเตือนผ่านอีเมล (SMTP) และ LINE (Messaging API) — เปิด/ปิดได้

## ความต้องการ
- Node.js เวอร์ชัน 18 ขึ้นไป (แนะนำ 20 LTS)
- ไม่ต้องติดตั้งฐานข้อมูลแยก ไม่มี native module ต้องคอมไพล์

## ติดตั้งและรัน (บนเครื่อง/VPS)
```bash
cd leave-server
npm install
cp .env.example .env          # แล้วแก้ค่า โดยเฉพาะ ADMIN_PASSWORD
npm start                     # เปิดที่ http://localhost:3000
```
เข้าใช้งานครั้งแรก ระบบจะโหลดรายชื่อเริ่มต้น 110 นายจาก `data/seed.json` ให้อัตโนมัติ

## นำเข้า/อัปเดตรายชื่อจาก Excel
วิธีที่ 1 — ผ่านหน้าเว็บ: เข้าสู่ระบบผู้ดูแล → เมนู "นำเข้า Excel" → เลือกไฟล์ .xlsx
วิธีที่ 2 — ผ่านบรรทัดคำสั่ง:
```bash
node import_roster.js "รายชื่อข้าราชการตำรวจ สภ.ดอนหัวฬ่อ.xlsx"
```
รูปแบบไฟล์: คอลัมน์ = ลำดับ | ยศ ชื่อ สกุล | ตำแหน่ง | โทรศัพท์ | นามเรียกขาน
โดยแถวชื่อกลุ่มงาน (เช่น "งานจราจร (19 นาย)") ให้อยู่คอลัมน์แรกและไม่มีเลขลำดับ
การนำเข้าจะคงใบลาเดิมและอีเมล/LINE เดิมไว้ (จับคู่ตามชื่อ)

## การกำหนดสิทธิ์อัตโนมัติ (จากตำแหน่ง)
- ตำแหน่งขึ้นต้น "ผกก."           → ผกก. (อนุมัติขั้นสุดท้าย)
- ตำแหน่งขึ้นต้น "รอง ผกก."       → รอง ผกก.
- ตำแหน่งขึ้นต้น "สวป." หรือ "สว." → หัวหน้างาน
- อื่น ๆ                          → เจ้าหน้าที่
หากกลุ่มงานใดไม่มีผู้มีตำแหน่งหัวหน้า ระบบจะใช้คนแรกของกลุ่มเป็นหัวหน้ารักษาการ
ปรับแก้ได้ที่เมนู "สายการอนุมัติ" ในหน้าผู้ดูแล

## Deploy บน Cloud/VPS (แนะนำ Ubuntu)
```bash
# 1) ติดตั้ง Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 2) อัปโหลดโฟลเดอร์ leave-server แล้ว
cd leave-server && npm install --omit=dev
cp .env.example .env && nano .env      # ตั้ง ADMIN_PASSWORD และพอร์ต

# 3) ให้รันค้างด้วย PM2 (รีสตาร์ทอัตโนมัติเมื่อเครื่องรีบูต)
sudo npm install -g pm2
pm2 start server.js --name leave
pm2 save && pm2 startup
```
แนะนำวาง Nginx เป็น reverse proxy หน้าเว็บ และเปิด HTTPS ด้วย Let's Encrypt (certbot)
ตัวอย่าง Nginx:
```
server {
  server_name leave.example.go.th;
  location / { proxy_pass http://127.0.0.1:3000; proxy_set_header Host $host; }
}
```

## การแจ้งเตือน
- อีเมล: ตั้ง `EMAIL_ENABLED=1` และค่า SMTP ใน `.env` (Gmail ต้องใช้ App Password)
  แล้วใส่อีเมลของแต่ละคนในเมนูจัดการรายชื่อ
- LINE: บริการ LINE Notify เดิมถูกยกเลิกแล้ว จึงใช้ LINE Messaging API แทน
  สร้าง channel ที่ https://developers.line.biz นำ Channel Access Token ใส่ `LINE_TOKEN`
  ตั้ง `LINE_ENABLED=1` และใส่ LINE userId ของแต่ละคน (หรือ `LINE_DEFAULT_TO` เป็นกลุ่มกลาง)

## การสำรองข้อมูล
ข้อมูลทั้งหมดอยู่ในไฟล์เดียว: `data/db.json` — สำรองไฟล์นี้เป็นระยะ (เช่น cron รายวัน)

## โครงสร้างไฟล์
```
leave-server/
  server.js         เซิร์ฟเวอร์หลัก + API + หน้าพิมพ์
  config.js         การตั้งค่า (อ่านจาก .env)
  db.js             ที่เก็บข้อมูล JSON (atomic write)
  roster.js         อ่าน Excel + กำหนดสิทธิ์/สายอนุมัติ
  notify.js         แจ้งเตือนอีเมล/LINE
  import_roster.js  นำเข้ารายชื่อผ่าน CLI
  public/index.html หน้าเว็บ (SPA)
  data/seed.json    รายชื่อเริ่มต้น
  data/db.json      ข้อมูลใช้งานจริง (สร้างอัตโนมัติ)
```

## ความปลอดภัย (ควรทำก่อนใช้จริง)
- เปลี่ยน `ADMIN_PASSWORD` เป็นรหัสที่คาดเดายาก
- เปิดใช้ HTTPS เสมอเมื่อรันบนอินเทอร์เน็ต
- จำกัดการเข้าถึงพอร์ตเซิร์ฟเวอร์ผ่าน firewall ให้เข้าได้เฉพาะผ่าน Nginx
